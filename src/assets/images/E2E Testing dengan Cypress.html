Berikut adalah contoh kode E2E testing menggunakan Cypress untuk memastikan bahwa tautan pada navbar berfungsi dengan benar dan fitur toggleDarkMode berfungsi:

javascript
describe('Navbar E2E Tests', () => {
  beforeEach(() => {
    cy.visit('/'); // Ganti dengan URL halaman yang sesuai
  });

  it('should navigate to the correct pages when links are clicked', () => {
    // Tes link ke halaman "Blog"
    cy.get('a[href="/blog"]').click();
    cy.url().should('include', '/blog');

    // Tes link ke halaman "About"
    cy.get('a[href="/about"]').click();
    cy.url().should('include', '/about');

    // Tes link ke halaman "Newsletter"
    cy.get('a[href="/newsletter"]').click();
    cy.url().should('include', '/newsletter');
  });

  it('should toggle dark mode when the dark mode button is clicked', () => {
    // Tes kondisi awal apakah tema terang (light mode)
    cy.get('html').should('not.have.class', 'dark');

    // Klik tombol dark mode
    cy.get('.flex .relative').click();

    // Pastikan tema berubah ke gelap (dark mode)
    cy.get('html').should('have.class', 'dark');

    // Klik lagi untuk kembali ke terang
    cy.get('.flex .relative').click();

    // Pastikan tema kembali ke terang
    cy.get('html').should('not.have.class', 'dark');
  });

  it('should toggle the navbar visibility when the hamburger menu is clicked', () => {
    // Pastikan navbar tersembunyi di perangkat mobile
    cy.get('nav').should('have.class', '-translate-y-full');

    // Klik tombol hamburger untuk membuka navbar
    cy.get('button[aria-label="Toggle menu"]').click();

    // Pastikan navbar muncul
    cy.get('nav').should('not.have.class', '-translate-y-full');

    // Klik tombol hamburger lagi untuk menutup navbar
    cy.get('button[aria-label="Toggle menu"]').click();

    // Pastikan navbar kembali tersembunyi
    cy.get('nav').should('have.class', '-translate-y-full');
  });

  it('should close the navbar when a link is clicked', () => {
    // Klik tombol hamburger untuk membuka navbar
    cy.get('button[aria-label="Toggle menu"]').click();

    // Klik link "Blog"
    cy.get('a[href="/blog"]').click();

    // Pastikan navbar tertutup setelah link diklik
    cy.get('nav').should('have.class', '-translate-y-full');
  });
});

Penjelasan:

  1. Tes link pada navbar:

    • Menggunakan cy.get('a[href="/link"]').click() untuk mengecek apakah klik pada link membawa pengguna ke halaman yang tepat.

  2. Tes fitur dark mode:

    • Menggunakan cy.get('html') untuk memeriksa apakah elemen html memiliki kelas dark yang menunjukkan tema gelap, dan melakukan klik pada tombol dark mode untuk memastikan bahwa tema berubah.

  3. Tes toggle navbar:

    • Memeriksa apakah navbar disembunyikan atau ditampilkan ketika tombol hamburger diklik menggunakan cy.get('nav') dan memeriksa kelas transformasi CSS (translate-y-0 dan -translate-y-full).

  4. Tes menutup navbar setelah klik link:

    • Setelah navbar terbuka, klik link di navbar untuk memastikan navbar tertutup secara otomatis.

1. Mengubah Ukuran Layar untuk Memastikan Tombol Hamburger Muncul

Anda dapat menggunakan cy.viewport() untuk mengubah ukuran layar sebelum mengklik tombol hamburger. Ini akan memastikan bahwa tombol hamburger muncul (untuk ukuran layar kecil) saat pengujian dilakukan.

Contoh cara mengubah ukuran layar ke ukuran mobile:

javascript
it('should toggle the navbar visibility when the hamburger menu is clicked', () => {
  // Mengubah ukuran layar ke ukuran mobile (misalnya 375x667 untuk mobile)
  cy.viewport(375, 667); // Ganti dengan ukuran sesuai kebutuhan Anda

  // Tes bahwa navbar tersembunyi pada ukuran layar kecil
  cy.get('nav').should('have.class', '-translate-y-full');

  // Klik tombol hamburger untuk membuka navbar

  cy.get('nav').should('have.class', '-translate-y-full');
});

2. Klik dengan { force: true }

Jika Anda ingin tetap menggunakan ukuran layar saat ini dan tidak mengubah viewport, Anda bisa menggunakan opsi { force: true } untuk memaksa klik pada elemen meskipun elemen tersebut tidak terlihat.

Namun, ini bukan solusi yang ideal karena Anda akan memaksa klik pada elemen yang seharusnya tidak dapat diakses. Solusi ini sebaiknya hanya digunakan jika Anda yakin tombol tersebut akan muncul atau dapat berfungsi dengan baik.

Contoh penggunaan { force: true }:

javascript
it('should toggle the navbar visibility when the hamburger menu is clicked', () => {
  // Klik tombol hamburger meskipun tidak terlihat
  cy.get('button[aria-label="Toggle menu"]').click({ force: true });

  // Pastikan navbar muncul setelah klik
  cy.get('nav').should('not.have.class', '-translate-y-full');

  // Klik tombol hamburger lagi untuk menutup navbar
  cy.get('button[aria-label="Toggle menu"]').click({ force: true });

  // Pastikan navbar kembali tersembunyi
  cy.get('nav').should('have.class', '-translate-y-full');
});

Jika Anda menggunakan cara pertama atau kedua, pastikan Anda memilih pendekatan yang sesuai dengan skenario pengujian Anda. Mengubah ukuran layar dengan cy.viewport() lebih disarankan karena ini mencerminkan perilaku pengguna yang sebenarnya pada berbagai perangkat.

Jika Anda ingin memastikan error muncul secara manual, Anda bisa mencoba mengatur error secara eksplisit dalam store menggunakan cy.window() untuk mengakses window dan dispatch action Redux.

4. Memverifikasi Penanganan Error di Redux

javascript
.addCase(addSubscribe.rejected, (state, action) => {
  state.loading = false;
  state.isSuccess = false;
  state.error = action.payload || "Something went wrong";  // Pastikan error dikirim dengan benar
});
